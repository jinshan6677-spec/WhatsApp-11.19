# 手动账户控制功能使用指南

## 简介

手动账户控制功能允许您自由地打开和关闭 WhatsApp 账号，而不是在应用启动时自动加载所有账号。这样可以：

- 💾 **节省系统资源** - 只运行需要的账号
- ⚡ **提高启动速度** - 应用启动更快
- 🎯 **灵活管理** - 按需打开和关闭账号
- 🔄 **保持会话** - 关闭后重新打开无需重新登录

## 快速开始

### 1. 应用启动

当您启动应用时，您会看到账号列表，但所有账号都处于"未启动"状态。

![账号列表](../resources/screenshots/account-list.png)

### 2. 打开账号

点击账号旁边的 **▶ 打开** 按钮来启动该账号。

- 按钮会变为"加载中..."指示器
- 几秒钟后，账号会显示为"已连接"状态
- 按钮会变为 **⏹ 关闭** 按钮

### 3. 关闭账号

点击 **⏹ 关闭** 按钮来停止该账号。

- 账号会立即关闭
- 会话数据会被保留
- 按钮会变回 **▶ 打开** 按钮

### 4. 重新打开账号

再次点击 **▶ 打开** 按钮，账号会恢复之前的登录状态，无需重新扫描二维码。

## 功能详解

### 账号状态

每个账号可以处于以下状态之一：

| 状态 | 图标 | 说明 | 可用操作 |
|------|------|------|---------|
| **未启动** | ⚪ 灰色 | 账号未运行 | 打开 |
| **加载中** | 🔵 蓝色 + 动画 | 正在加载 WhatsApp Web | 等待 |
| **已连接** | 🟢 绿色 | 账号正在运行 | 关闭 |
| **错误** | 🔴 红色 | 加载失败 | 重试 |

### 打开账号

**操作步骤：**
1. 在账号列表中找到要打开的账号
2. 点击 **▶ 打开** 按钮
3. 等待加载完成（通常 2-5 秒）
4. 账号状态变为"已连接"

**注意事项：**
- 首次打开需要扫描二维码登录
- 后续打开会自动恢复登录状态
- 同时最多可以打开 10 个账号

### 关闭账号

**操作步骤：**
1. 找到正在运行的账号
2. 点击 **⏹ 关闭** 按钮
3. 账号立即关闭，状态变为"未启动"

**注意事项：**
- 关闭账号会释放内存资源
- 会话数据会被保留
- 聊天记录和登录状态不会丢失

### 自动启动

您可以配置某些账号在应用启动时自动打开。

**设置自动启动：**
1. 点击账号旁边的 **⚙️ 设置** 按钮
2. 在账号设置对话框中，勾选"自动启动"选项
3. 点击"保存"
4. 下次启动应用时，该账号会自动打开

**取消自动启动：**
1. 打开账号设置
2. 取消勾选"自动启动"选项
3. 保存设置

### 错误处理

如果账号打开失败，会显示错误状态。

**处理错误：**
1. 查看错误提示信息
2. 点击 **⟳ 重试** 按钮
3. 如果仍然失败，检查网络连接
4. 必要时重启应用

**常见错误：**
- **网络连接失败** - 检查网络连接
- **加载超时** - 重试或重启应用
- **达到账号限制** - 关闭其他账号后再试

## 使用场景

### 场景 1: 工作时间只使用工作账号

**问题：** 您有多个账号，但工作时只需要使用工作账号。

**解决方案：**
1. 启动应用
2. 只打开工作账号
3. 其他账号保持关闭状态
4. 节省资源，提高性能

### 场景 2: 临时使用某个账号

**问题：** 您需要临时查看某个不常用账号的消息。

**解决方案：**
1. 打开该账号
2. 查看和回复消息
3. 使用完毕后关闭账号
4. 释放资源

### 场景 3: 多账号轮流使用

**问题：** 您需要在多个账号之间切换，但不想同时运行所有账号。

**解决方案：**
1. 打开当前需要的账号
2. 使用完毕后关闭
3. 打开下一个需要的账号
4. 保持低内存占用

### 场景 4: 自动启动常用账号

**问题：** 您有几个常用账号，希望应用启动时自动打开。

**解决方案：**
1. 为常用账号启用"自动启动"
2. 其他账号保持手动启动
3. 应用启动时自动打开常用账号
4. 按需打开其他账号

## 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl/Cmd + 1-9` | 切换到第 1-9 个账号 |
| `Ctrl/Cmd + Tab` | 切换到下一个账号 |
| `Ctrl/Cmd + Shift + Tab` | 切换到上一个账号 |
| `Ctrl/Cmd + R` | 重新加载当前账号 |

**注意：** 快捷键只能切换到已打开的账号。

## 性能优化建议

### 内存管理

- **建议同时打开的账号数：** 3-5 个
- **最大同时打开数：** 10 个
- **每个账号内存占用：** 约 150-200 MB

### 最佳实践

1. **只打开需要的账号**
   - 不使用的账号保持关闭状态
   - 定期关闭不活跃的账号

2. **使用自动启动**
   - 为常用账号启用自动启动
   - 不常用账号手动打开

3. **定期重启应用**
   - 长时间运行后重启应用
   - 释放累积的内存

4. **监控资源使用**
   - 注意系统内存使用情况
   - 必要时关闭部分账号

## 故障排除

### 问题 1: 账号打开失败

**症状：** 点击"打开"按钮后显示错误状态

**解决方法：**
1. 检查网络连接
2. 点击"重试"按钮
3. 重启应用
4. 检查防火墙设置

### 问题 2: 账号关闭后无法重新打开

**症状：** 关闭账号后，再次打开失败

**解决方法：**
1. 等待几秒钟后重试
2. 重启应用
3. 检查会话数据目录权限

### 问题 3: 自动启动不工作

**症状：** 启用自动启动后，应用启动时账号没有自动打开

**解决方法：**
1. 检查账号设置中的"自动启动"选项是否勾选
2. 重启应用
3. 检查账号配置文件

### 问题 4: 达到账号数量限制

**症状：** 尝试打开账号时提示"达到最大账号数量限制"

**解决方法：**
1. 关闭一些不需要的账号
2. 等待关闭完成后再打开新账号
3. 考虑增加最大账号数量限制（高级用户）

### 问题 5: 内存占用过高

**症状：** 应用占用大量内存

**解决方法：**
1. 关闭不需要的账号
2. 重启应用
3. 减少同时打开的账号数量
4. 升级系统内存

## 高级功能

### 配置最大账号数量

默认情况下，最多可以同时打开 10 个账号。如果需要修改：

1. 打开配置文件（高级用户）
2. 修改 `maxConcurrentViews` 设置
3. 重启应用

**注意：** 增加限制会增加内存占用。

### 会话数据管理

会话数据存储在以下位置：
- **Windows:** `%APPDATA%/whatsapp-desktop-translation/profiles/`
- **Mac:** `~/Library/Application Support/whatsapp-desktop-translation/profiles/`
- **Linux:** `~/.config/whatsapp-desktop-translation/profiles/`

每个账号有独立的会话数据目录。

### 清理孤立数据

应用会自动清理已删除账号的会话数据。如果需要手动清理：

1. 关闭应用
2. 删除对应账号的会话数据目录
3. 重启应用

## 常见问题 (FAQ)

### Q: 关闭账号后会丢失聊天记录吗？

**A:** 不会。关闭账号只是停止运行，所有数据（包括聊天记录、登录状态）都会保留。重新打开时会自动恢复。

### Q: 可以同时打开多少个账号？

**A:** 默认最多 10 个。实际建议根据您的系统配置，同时打开 3-5 个账号以获得最佳性能。

### Q: 自动启动会影响应用启动速度吗？

**A:** 会有轻微影响。每个自动启动的账号需要 2-5 秒加载时间。建议只为常用账号启用自动启动。

### Q: 关闭账号后需要重新扫码登录吗？

**A:** 不需要。会话数据会被保留，重新打开时会自动恢复登录状态。

### Q: 如何知道账号是否正在运行？

**A:** 查看账号旁边的状态指示器：
- 🟢 绿色 = 正在运行
- ⚪ 灰色 = 未运行
- 🔵 蓝色 = 加载中
- 🔴 红色 = 错误

### Q: 可以在账号运行时编辑账号设置吗？

**A:** 可以，但某些设置需要重新打开账号才能生效。

### Q: 如何备份账号数据？

**A:** 备份会话数据目录即可。位置见"高级功能 > 会话数据管理"。
